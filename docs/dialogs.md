# Диалоги — как наполнять

Диалог — это цепочка реплик, которая всплывает по триггеру: экран затемняется, появляется
баббл с портретом говорящего и текстом, клик по экрану листает реплики, после последней —
игрока отпускают (продолжается обычный ход игры).

## Где редактировать

Контент живёт в Google-таблице (лист **dialogs**, `gid=915816125`):
<https://docs.google.com/spreadsheets/d/1EUfW2SPLCzWxCaqrYfPNz27JhQMIxda1Z9DLqmIEQE4/edit?gid=915816125>

После правок в Unity: **меню `Mimic → Import Configs from Google Sheets`** — скачает все
конфиги, включая `dialogs.csv.txt`. Файл `dialogs.csv.txt` перезаписывается целиком, руками
его править не нужно (правки затрутся импортом).

> Таблица должна быть расшарена «Anyone with the link», иначе импорт получит HTML вместо CSV.

## Формат таблицы

Три колонки:

| колонка | смысл |
|---------|-------|
| `trigger` | ключ, по которому запускается цепочка. **Непустой** — начинает НОВУЮ цепочку. **Пустой** — добавляет реплику в текущую (предыдущую) цепочку. |
| `text` | текст реплики (одна строка баббла). |
| `icon` | id портрета говорящего (см. ниже). |

Пример:

```csv
trigger,text,icon
start_day_1,Привет! Добро пожаловать в подземелье.,master
,Сегодня к тебе заглянут приключенцы. Не оплошай.,master
end_day_1,Посмотрим как ты поработал!,master
start_day_2,Надо сбежать от этого узурпатора.,mimic
```

Здесь `start_day_1` — цепочка из ДВУХ реплик (вторая строка с пустым `trigger`
продолжает её), `end_day_1` и `start_day_2` — по одной реплике.

## Триггеры

Триггеры формирует код игры. Сейчас поддержаны:

| триггер | когда срабатывает |
|---------|-------------------|
| `start_day_{N}` | началась N-я игровая неделя/день (в т.ч. самый первый старт и переигровка дня). |
| `end_day_{N}` | игрок нажал «Завершить день» — перед появлением экрана Властелина. |

`{N}` — номер дня из `day.csv` (колонка `day`): `start_day_1`, `end_day_2`, …

- Триггер срабатывает **каждый раз** при наступлении события (в т.ч. при переигровке дня).
- Если для триггера нет ни одной строки — диалог просто пропускается, игра идёт дальше.

## Портреты (`icon`)

`icon` — это id префаба-портрета. Загрузка ищет префаб в таком порядке:

1. `Resources/Art/Portraits/{icon}.prefab`
2. `Resources/Art/Adventurers/{icon}.prefab` (реюз портретов приключенцев)

То есть `icon = warrior` подхватит портрет воина из `Art/Adventurers`, а для рассказчиков
(`master`, `mimic` и т.п.) клади префабы в `Resources/Art/Portraits/`.

Если префаб не найден — баббл покажет заглушку (текст всё равно отобразится). Так что можно
писать диалоги до того, как готов арт.

## Чек-лист «добавить новый диалог»

1. В листе **dialogs** добавить строку с `trigger` (или несколько строк: первая с триггером,
   следующие — с пустым `trigger` для продолжения цепочки).
2. Указать `icon` существующего портрета (или нового — тогда положить префаб в
   `Resources/Art/Portraits/`).
3. В Unity: `Mimic → Import Configs from Google Sheets`.
4. Запустить игру и проверить, что диалог всплывает в нужный момент.
