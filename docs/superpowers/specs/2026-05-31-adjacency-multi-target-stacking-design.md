# Adjacency: блочный формат, мульти-таргет, вайлдкард и стак

**Дата:** 2026-05-31
**Статус:** дизайн утверждён (ревизия 2 — новый блочный формат)

## Цель

Переделать систему Adjacency на единое поле конфига с блочной грамматикой:

1. **Один предмет — несколько блоков «таргеты → эффекты».** Разные соседи дают
   разные эффекты.
2. **Несколько таргетов на один набор эффектов** (через `,`).
3. **Вайлдкард `*`** — «все остальные» соседи (чей id не назван явно ни в одном
   блоке этого предмета).
4. **Стакающиеся / нестакающиеся эффекты** — суффикс `*` на эффекте.
5. Все эффекты складываются **аддитивно** и применяются к базе один раз.

## Формат конфига (`loot.csv`)

Старый столбец `adjacencyTarget` **удаляется**. Остаётся один столбец
`adjacencyEffect` с грамматикой:

```
<block> (';' <block>)*
<block>   := <targets> '|' <effects>
<targets> := <id|*> (',' <id|*>)*
<effects> := <effect> (',' <effect>)*
<effect>  := <type> ':' <sign?> <n> '%' '*'?
```

- `;` — разделитель **блоков** (хвостовой `;` допускается).
- `|` — разделяет таргеты и эффекты внутри блока.
- `,` — список таргетов **или** список эффектов (по сторонам от `|`).
- `*` слева от `|` — **вайлдкард-таргет** (все неназванные соседи). Смешивать
  `*` с конкретными id в одном блоке нельзя (ошибка парсинга).
- `*` в конце эффекта — **стакающийся** (за каждый прилегающий инстанс).
- Знак значения опционален: `gold:5%` = `gold:+5%`.
- Пробелы вокруг токенов и `:` игнорируются (`gold: -50%` валидно).
- **Переносы строк игнорируются** (`\n`/`\r` нормализуются в пробел перед
  парсингом). Один и тот же текст парсится одинаково многострочным (удобно
  оформлять в Google Docs) и схлопнутым в одну строку. В самом `loot.csv` поле
  остаётся **однострочным** (CsvLoader не поддерживает многострочные поля) —
  при переносе из Google Docs блоки склеиваются в одну строку.

### Пример

```
adjacencyEffect: "hat | gold:+50% , acid:+50% ; sheath , frog | gold:-50% ; *|gold:5%"
```

- рядом с `hat` → +50% к цене и +50% к стоимости переваривания;
- рядом с `sheath` или `frog` → −50% к цене;
- рядом с любым прочим (не hat/sheath/frog) предметом → +5% к цене.

### CSV-кавычки

Грамматика содержит `,`, совпадающий с разделителем CSV, поэтому поле с
несколькими блоками/эффектами/таргетами **обязательно берётся в кавычки**:
`"...;..."`. `CsvLoader` поддерживает кавычки и экранирование `""` (но не
многострочные поля — весь блок пишется в одну строку). Поля с одним
`target|effect` без запятых можно не кавычить (`sword|gold:+50%`).

## Модель данных

`AdjacencyEffect` (struct) — добавляется `Stackable`, парсинг одного эффекта
выносится в публичный `Parse(token)` (старый `ParseList` удаляется — `;` больше
не разделитель эффектов):

```csharp
public struct AdjacencyEffect
{
    public EffectType Type;     // Gold | Acid
    public float Multiplier;    // +0.5 = +50%
    public bool Stackable;      // '*' суффикс
    public static AdjacencyEffect Parse(string token);
}
```

`AdjacencyRule` (новый struct) — один блок:

```csharp
public struct AdjacencyRule
{
    public string[] Targets;          // id соседей; пусто при Wildcard
    public bool Wildcard;             // таргет '*'
    public AdjacencyEffect[] Effects; // эффекты блока
    public static AdjacencyRule[] ParseRules(string raw);
}
```

`LootData`:

- удаляются `AdjacencyTarget` (string) и `AdjacencyEffects` (AdjacencyEffect[]);
- добавляется `AdjacencyRule[] AdjacencyRules` (never null; пустой = нет свойства).

## Логика resolver

`AdjacencyResolver.Resolve` принимает `Func<T, AdjacencyRule[]> adjacencyRulesOf`
(вместо двух старых делегатов).

Для каждого предмета:

1. Собрать множество **названных** таргетов предмета — объединение `Targets`
   всех его невайлдкард-блоков (нужно, чтобы вайлдкард бил только по неназванным).
2. Аккумуляторы `sumGold = 0`, `sumAcid = 0`.
3. Для каждого блока предмета:
   - посчитать `count` = число **различных** прилегающих инстансов (4-соседство
     по ребру, `GetEdgeNeighbors` возвращает HashSet различных инстансов):
     - невайлдкард: сосед, чей `id ∈ Targets`;
     - вайлдкард: сосед, чей `id ∉ named`.
   - если `count == 0` — пропустить блок.
   - для каждого эффекта блока: `times = Stackable ? count : 1`; прибавить
     `Multiplier * times` к `sumGold`/`sumAcid` по типу.
4. Применить **аддитивно, один раз** к базе:
   - `EffectiveGold = round(baseGold * (1 + sumGold))`, пол 0;
   - `EffectiveAcid = round(baseAcid * (1 + sumAcid))`, пол 1.

Отрицательные стаки суммируются как есть (без клампа вклада на −100%), итог
ограничен только полом.

### Что НЕ меняется

- Исходящий `NeighborGoldPct` (какашка) — как есть, применяется после,
  мультипликативно к итоговому золоту соседей.
- `TotalGold` считается после всех эффектов.
- 4-соседство по ребру; self-сосед игнорируется.

## Изменение поведения

Старый resolver применял эффекты компаундом; новый — аддитивно (`+50% + +50% =
+100%`). Существующие одиночные правила (`shield`, `gem`, `barracuda`, `poop`)
ведут себя идентично — разница видна только при 2+ множителях одного типа.

## Тестирование

Автотестов нет (game jam, asmdef/CI запрещены). Верификация — прогон ассертов
через Unity MCP `execute_code` после компиляции. Контрольные кейсы:

- парсинг блочной строки из примера → 3 блока (hat; sheath/frog; wildcard).
- тот же текст с переносами строк (как в Google Docs) → те же 3 блока.
- `gold:5%` без знака → +0.05; `gold:+50%*` → Stackable.
- блок без `|` → FormatException.
- стак `gold:+50%*`, 2 hat → +100%; нестак `gold:+50%`, 2 hat → +50%.
- мульти-таргет в одном блоке `hat,sheath|gold:+50%` (нестак), рядом hat+sheath →
  +50% (один раз).
- два блока `hat|...;sheath|...` рядом hat+sheath → суммируются.
- вайлдкард: `frog|gold:-50%;*|gold:+10%`, рядом frog + bread + gem →
  −50% (frog) + 10% (двое прочих, нестак) = −40%.
- отрицательный стак `frog|gold:-50%*`, 3 frog → −150% → золото 0.
- `LootCatalog.Load()` грузит обновлённый `loot.csv.txt` без исключений.
