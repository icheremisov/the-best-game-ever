# Дизайн: голубой (среднесрочный) луп

**Дата:** 2026-05-29
**Источник:** «Декомпоз корлупа (голубая часть)» (PDF, вне репо)
**Контекст:** game jam «Is this a mimic?», Unity 6, URP 2D. Скорость важнее чистоты, без архитектур на будущее.

## Цель

Спроектировать всю среднесрочную петлю («День») как обёртку вокруг уже готового
зелёного лупа (обработка приключенцев): экран Властелина, награда/наказание,
цепочка дней, концовки. Боссфайт — стаб.

## Что уже есть (не трогаем по сути)

- `GameFlow` — очередь приключенцев на день, intro-попап, выкладка лута, переваривание за ЖС, пересчёт золота с adjacency.
- `GameResources` — HP / ЖС / золото-в-мимике / квота.
- Сетки (`MimicGrid`, `AdventurerGrid`), drag&drop, tooltip, контекстное меню.
- Конфиги `day/adventurers/loot.csv` (в `Resources/Configs/`), `AdjacencyResolver`.
- `EndOfDayPopup` (минимальный win/lose) — расширяем.

## §1. Модель состояния и ресурсов

`GameResources` переживает смену дня (объект не пересоздаётся при переходе):

| Поле | Поведение |
|------|-----------|
| `CurrentHp` | переносится между днями |
| `CurrentAcid` | восстанавливается до `StartAcid` в начале каждого дня |
| `HeldGold` | живая сумма золота лута в сетке мимика (с adjacency) = текущий `CurrentGoldInMimic` |
| `BankedGold` | **новое.** Золото, сданное в корзину Властелину. Копится навсегда, не тратится |
| `TotalGold` | `HeldGold + BankedGold`. Против него проверяется квота дня И порог выкупа |

- Лут в сетке мимика **не очищается** между днями.
- Снапшот состояния на входе в день (HP, BankedGold, содержимое сетки мимика) — для «Переиграть день».

## §2. Стейт-машина дня (в `GameFlow`)

```
enum DayPhase { Adventurers, Overlord, RewardChoice, Transition }
```

- **Adventurers** — как сейчас: очередь приключенцев, выкладка/переваривание.
  По завершению очереди кнопка → «Завершить день» → переход в `Overlord`.
- **Overlord** — экран Властелина (корзина + воровство + «Подвести итог дня»).
- **RewardChoice** — выбор 1 из 3 (награда или наказание).
- **Transition** — выбор игрока: следующий день / выкуп / вызов (§6).

## §3. Экран Властелина (`OverlordPopup` / режим HUD)

Правая область (где был грид приключенца) превращается в **корзину**. Игрок может:

- **Перетащить лут в корзину** → предмет уходит из мимика; его золото (с учётом
  adjacency на момент сдачи) прибавляется к `BankedGold`; место в мимике освобождается.
  Это размен «золото вместо места» — отдаёшь лишнее, чтобы не лопнуть.
  Предметы-наказания с флагом «нельзя вернуть» в корзину не кладутся.
- **Тултип** над «Подвести итог дня»: `[есть]/[надо]` золота, сколько HP потеряется,
  текст «Последует награда» / «Последует наказание».
- **Атака / КУСЬ** — перетаскивание атакующего предмета на портрет Властелина или
  кнопка КУСЬ → переход в боссфайт-**стаб** (§6).

При нажатии **«Подвести итог дня»**:
1. **Воровство:** Властелин крадёт 1 случайный предмет со свободной верхней гранью.
   *Свободная верхняя грань* = у предмета есть клетка, у которой все клетки строго
   вверх до верхней границы сетки пусты. Если таких предметов нет — не крадёт.
2. **Подсчёт:** `TotalGold` vs `quota` дня.
3. Если `TotalGold < quota`: урон `dmg = (quota − TotalGold) × goldDamageMult`,
   анимация полоски HP. Если `HP ≤ 0` — концовка смерти (§6). Иначе → `RewardChoice`.

## §4. Награда / Наказание (`RewardChoicePopup`)

Корзина справа меняется на 3 предмета. `TotalGold ≥ quota` → пул наград, иначе → пул
наказаний. Игрок забирает **1** → предмет кладётся в сетку мимика (first-fit).
Если не влезает → концовка «лопнул» (consistent с lose-условием). Затем → `Transition`.

Предметы добавляются в `loot.csv`, новая колонка `category = normal | reward | punish`.

**Награды:**
- Мимик-бургер — при переваривании восстанавливает HP.
- Бутылка кислоты — при переваривании восполняет ЖС; *урон врагу* (инертно при стабе).
- Огне-кола — огненная отрыжка / *урон врагу* (инертно при стабе).

**Наказания:**
- Гиря 60кг — *огромный урон при переваривании*, занимает много места, нельзя в корзину.
- Какашка — +50% урона рядом с сердцем, +50% ЖС рядом с желудком, −50% золота соседям, нельзя в корзину.
- Масса клея — небольшой урон при переваривании; склеивает соседей (переносятся вместе), нельзя в корзину.
- Зуб с кариесом — не наносит урон при переваривании, стоит максимум ЖС; *блокирует КУСЬ*, *−50% исходящего урона* (инертно при стабе).

> ⚠️ **Джем-деферрал.** Эффекты, завязанные на боевой энкаунтер (урон врагу,
> отрыжка, блок КУСЬ, исходящий урон) при стабе боссфайта **инертны** — реализуем
> только инвентарные эффекты: HP/ЖС при переваривании, занимаемое место,
> adjacency-модификаторы цены/ЖС, склейка клеем.
>
> **Сердце и желудок** как фиксированные объекты в сетке мимика нужны для adjacency
> «рядом с сердцем / рядом с желудком» (есть на мокапах ТЗ) — часть работы.

## §5. Цепочка дней (config-driven)

`day.csv` уже многострочный. Переход `Transition → следующий день`:
`day++` → грузим следующую строку → `StartDay()` (ЖС в фул; HP, лут, `BankedGold`
сохраняются) → фаза `Adventurers`. **Без перезагрузки сцены.**

Новые колонки `day.csv`: `goldDamageMult`, `ransomGold` (порог выкупа).
Последний день = последняя строка файла.

## §6. Концовки и триггеры финала (`Transition`)

После награды показываем выбор (доступность кнопок по условиям) — **доступно после
любого дня** (как в ТЗ):

- **[Следующий день]** — если остались дни.
- **[Выкупить себя]** — если `TotalGold ≥ ransomGold` → концовка «свободный мимик».
- **[Бросить вызов]** (КУСЬ) → **стаб боссфайта**: заглушка-экран → возврат в меню.
  Полноценные концовки вызова («умер счастливым» / «король-мимик») — в красном/жёлтом лупе.

**Поражения** (`EndOfDayPopup`, тексты из ТЗ):
- Лопнул (нет места) / HP = 0.
- Кнопки: «Переиграть день» (откат к снапшоту начала проваленного дня) / «В меню».
