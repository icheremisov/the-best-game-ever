# Конфиги игры — спецификация для Google-таблиц

Документ для геймдизайнеров: как устроены CSV-конфиги и как вести их в Google-таблице.

## Как это работает

1. Каждый конфиг — отдельная вкладка/таблица в Google Sheets.
2. Экспорт: **Файл → Скачать → CSV** (текущий лист).
3. Скачанный файл переименовать в `<имя>.csv.txt` и положить в
   `Game/Assets/Resources/Configs/` (поверх существующего).
   Имена строго: `loot.csv.txt`, `adventurers.csv.txt`, `day.csv.txt`.
4. В Unity: правый клик по папке `Configs` → **Reimport** (или просто фокус
   редактора). Изменения подхватятся при следующем входе в Play.

### Общие правила формата

- **Первая строка — заголовок** (имена колонок). Парсер её пропускает; важен
  ПОРЯДОК колонок, а не их названия. Не меняйте порядок и не вставляйте колонки
  в середину — дописывайте только в конец.
- **Пустые строки** игнорируются.
- **Разделитель — запятая.** Если внутри ячейки есть запятая (например, список
  таргетов `hat, magic_staff`), Google Sheets при экспорте сам обернёт ячейку в
  кавычки `"..."` — это корректно и парсится.
- **Дробные числа — только через точку** (`1.5`, не `1,5`). Если у вас русская
  локаль в Sheets, проверьте экспортированный CSV: должна быть точка.
- **Многострочные ячейки** разрешены (перенос строки внутри ячейки) — например,
  чтобы записать несколько adjacency-правил столбиком. При экспорте такая ячейка
  оборачивается в кавычки; парсер склеит строки.
- **Кодировка — UTF-8** (Google CSV по умолчанию). Кириллица поддерживается.
- **Незаполненные опциональные числовые ячейки** трактуются как `0`
  (см. пометку «опц.» у колонок). Обязательные ячейки пустыми оставлять нельзя.
- **Ссылки на id** (лут у приключенца, таргеты adjacency, приключенцы у дня)
  должны указывать на существующие id, иначе будет ошибка при загрузке.

## 1. `loot.csv` — предметы

20 колонок (по порядку). Хвостовые колонки (9–19) опциональны — недостающие читаются со значением по умолчанию, так что короткие строки валидны:

| # | Колонка | Тип | Обяз. | Значение |
|---|---------|-----|-------|----------|
| 0 | `id` | строка | да | Уникальный идентификатор предмета. Латиница/цифры. На него ссылаются приключенцы и adjacency. |
| 1 | `name` | строка | да | Отображаемое имя (в тултипе/попапах). Может быть пустым у фикстур. |
| 2 | `description` | строка | нет | Описание для тултипа («курсивом»). |
| 3 | `shape` | формат | да | Форма предмета в сетке. См. [Форма](#формат-shape). |
| 4 | `gold` | целое | да | Базовая цена (золото при сдаче). |
| 5 | `acidCost` | целое | да | Сколько желудочного сока (ЖС) стоит переварить. |
| 6 | `healOnDigest` | целое | да | +HP мимику при переваривании (бургер). |
| 7 | `adjacencyEffect` | формат | нет | Правила «ЕСЛИ РЯДОМ С …». См. [Adjacency](#формат-adjacency). Пусто = нет свойства. |
| 8 | `category` | enum | нет (опц., деф. `normal`) | `normal` / `reward` / `punish` / `fixture`. См. [Категории](#категории). |
| 9 | `acidRestoreOnDigest` | целое | опц. (0) | +ЖС при переваривании (кислота/мизим). |
| 10 | `damageOnDigest` | целое | опц. (0) | −HP мимику при переваривании (гиря/клей/какашка). Показывается в тултипе красным. |
| 11 | `canReturnToBasket` | 0/1 | опц. (1) | Можно ли сдать/вернуть в корзину. Наказания обычно `0`. |
| 12 | `glue` | 0/1 | опц. (0) | `1` = масса клея (склеивает соседей в группу). |
| 13 | `neighborGoldPct` | целое | опц. (0) | Модификатор золота соседям, в % (какашка: `-50`). |
| 14 | `attack` | целое | опц. (0) | **Бой:** урон по врагу при броске предмета в зону «Атаковать». `0` = предмет не бросается (возвращается на место). |
| 15 | `attackOnDigest` | целое | опц. (0) | **Бой:** урон врагу при переваривании во время боя (бесплатный бонус). |
| 16 | `goldOnDigestPct` | целое | опц. (0) | % от **базовой** цены (`gold`), который мимик отрыгнёт **монеткой** при переваривании. `0` = монетки нет. Монетка занимает 1 клетку и появляется на месте переваренного предмета. |

> **Арт предмета — не в конфиге.** Визуал (спрайт, масштаб, сдвиг, эффекты) задаётся
> арт-префабом `Assets/Resources/Art/Loot/{id}.prefab` (имя файла = `id` предмета).
> Префаб настраивается прямо в Unity. Если префаба нет — предмет рисуется цветной
> заглушкой с подписью. Подробнее см. [Арт лута](#арт-лута).

> Колонки 4–6 (`gold`, `acidCost`, `healOnDigest`)
> обязательны и должны содержать число (можно `0`), пустыми их оставлять нельзя.

> `goldOnDigestPct` считается от базовой цены, а не от цены с учётом adjacency.
> У наказаний/наград с `gold=0` монетки не будет (0% от 0). Чтобы предмет вообще
> не давал монетку, оставьте `0` (значение по умолчанию).

### Категории

| Значение | Смысл |
|----------|-------|
| `normal` | Обычный лут приключенцев. |
| `reward` | Предмет-награда: попадает в пул окна «Награда» при закрытой квоте дня. |
| `punish` | Предмет-наказание: пул окна «Наказание» при незакрытой квоте. |
| `fixture` | Сердце/желудок: нельзя двигать и переваривать. |

> Если в `loot.csv` нет ни одного `reward`/`punish` — окно награды/наказания
> будет пустым. Нужны минимум по нескольку каждой категории.

### Пример строки
```
sword,Меч,"Острый, но тупой",X|X|X|X,10,3,0,,normal,0,0,1,0,0,6,0,25
```
Меч: форма 1×4, цена 10, переварить за 3 ЖС, атака при броске 6,
при переваривании отрыгивает монетку на 25% от 10 = 2 золота.

### Арт лута

Внешний вид предмета **не задаётся в CSV** — это чисто визуальная вещь, её неудобно
крутить в гуглдоке. Вместо этого используется **арт-префаб**:

- Путь: `Assets/Resources/Art/Loot/{id}.prefab`, где `{id}` — `id` предмета из `loot.csv`.
- Загружается в рантайме по имени (`Resources.Load`). Если префаба нет — предмет
  рисуется цветной клеткой-заглушкой с подписью (имя), чтобы было видно, что арт не назначен.
- Префаб настраивается **прямо в Unity**: спрайт, масштаб, сдвиг, доп. эффекты/слои.

**Как движок использует префаб:**
- Создаётся обёртка размером с bounding box формы (в ориентации Deg0), центрированная
  по форме и **поворачивающаяся вместе с фигурой** при вращении.
- Префаб инстанцируется внутрь обёртки «как есть» (его локальный трансформ сохраняется).
- Базовый шаблон префаба — `Image` на всю обёртку (anchors stretch + `Preserve Aspect`):
  так арт автоматически вписывается в форму. Хочешь крупнее/сдвинуть — меняешь размер,
  `anchoredPosition`, `localScale` или сам спрайт прямо в префабе.
- Весь арт неинтерактивен (raycast выключается автоматически) — клики/тултип идут по клеткам.

**Фикстуры** (`heart`, `stomach`): рисуются только префабом, без клеток/подсветки/тултипа —
они «вшиты» в панель.

> Чтобы добавить арт новому предмету: положи спрайт в проект, создай префаб
> `Resources/Art/Loot/{id}.prefab` с `Image` этого спрайта (stretch + Preserve Aspect),
> настрой как надо. Готово — код подхватит его по `id`.

## 2. `adventurers.csv` — приключенцы

8 колонок:

| # | Колонка | Тип | Обяз. | Значение |
|---|---------|-----|-------|----------|
| 0 | `id` | строка | да | Уникальный id приключенца. На него ссылается `day.adventurerIds`. |
| 1 | `name` | строка | да | Имя (в интро-попапе). |
| 2 | `phrase` | строка | да | Реплика в интро-попапе. |
| 3 | `lootIds` | список | да | **Пул** лута приключенца — id предметов через `;`. Напр. `bow;crowbar;hat`. Все id должны существовать в `loot.csv`. Это набор, из которого случайно выдаётся лут (см. `budget`), а не гарантированный список. |
| 4 | `battle` | 0/1 | опц. (0) | **Бой:** `1` = с приключенцем нужно сразиться перед получением лута (кнопка «В бой»). |
| 5 | `hp` | целое | опц. (0) | **Бой:** здоровье приключенца. Используется только при `battle=1`. |
| 6 | `attack` | целое | опц. (0) | **Бой:** урон, который приключенец наносит мимику за ход. |
| 7 | `budget` | целое | опц. (0) | **Бюджет выдачи лута в клетках.** Приключенец случайно набирает предметы из `lootIds`, пока их суммарный размер (число занятых клеток формы) не достигнет бюджета. На каждом шаге из пула берётся случайный предмет, который ещё влезает в остаток бюджета (равновероятно среди подходящих по размеру, **с повторами** — один предмет может выпасть несколько раз). Цикл останавливается, когда ни один предмет пула не помещается в остаток. `0`/пусто => выдать **весь** `lootIds` целиком (старое поведение). |

### Пример
```
warrior,Воин,definitely not a mimic!,bow;crowbar;quiver;hat;magic_staff,0,0,0,24
dragon,Дракончик,попробуй проглоти МЕНЯ,diamond;rich_diadem;hat,1,12,2,14
```
`warrior` — обычный, выдаст ~24 клетки случайного лута из своего пула; `dragon` — боевой (hp 12, атака 2), бюджет 14 клеток.

## 3. `day.csv` — дни

10 колонок:

| # | Колонка | Тип | Обяз. | Значение |
|---|---------|-----|-------|----------|
| 0 | `day` | целое | да | Номер дня (для отображения). |
| 1 | `goldQuota` | целое | да | Квота золота, которую нужно набрать за день. |
| 2 | `startHp` | целое | да | Стартовое HP мимика (только в 1-й день; дальше HP переносится). |
| 3 | `startAcid` | целое | да | ЖС на старте дня (восполняется каждый день). |
| 4 | `adventurerIds` | список | да | Очередь приключенцев дня — id через `;`. Можно повторять. Напр. `warrior;rogue;dragon;mage;rogue`. |
| 5 | `goldDamageMult` | дробное | опц. (1) | Множитель урона за недобор квоты. Точка-разделитель: `1.5`. |
| 6 | `ransomGold` | целое | опц. (999999) | Сколько золота нужно, чтобы «выкупить себя» в конце. |
| 7 | `overlordHp` | целое | опц. (20) | **Бой:** HP Хозяина в финальном бою («Бросить вызов»). |
| 8 | `overlordAttack` | целое | опц. (2) | **Бой:** урон Хозяина за ход. |
| 9 | `biteDamage` | целое | опц. (2) | **Бой:** урон стандартной атаки «Кусь» (общий для всех боёв этого дня). |

### Пример
```
1,200,10,15,warrior;rogue;dragon;mage;rogue,1,200,20,2,5
```
День 1: квота 200, старт HP 10 / ЖС 15, очередь из 5 приключенцев (3-й — боевой
дракон), множитель урона 1, выкуп 200, Хозяин hp 20/атака 2, «Кусь» бьёт на 5.

## Формат `shape`

Сетка предмета. Строки разделяются `|`, в строке: `X` — занятая клетка, `.` — пустая.
Все строки одной длины.

| Пример | Значение |
|--------|----------|
| `X` | 1 клетка |
| `XX` | горизонтальный домино 1×2 |
| `X\|X\|X\|X` | вертикальная палка 4×1 |
| `XX\|XX` | квадрат 2×2 |
| `.X.\|XXX\|XXX` | «шляпа» (T-образная шапка) |
| `.X\|XX\|.X\|.X` | барракуда |

Правила:
- Только символы `X` и `.`.
- Каждая строка (между `|`) — одинаковой длины, иначе ошибка загрузки.
- Это форма ДО поворота; игрок может вращать предмет в игре.

## Формат `adjacency`

Свойства «ЕСЛИ РЯДОМ С [предмет]: модификатор». Поле может содержать несколько
правил.

### Грамматика
```
adjacency := блок (';' блок)*
блок      := таргеты '|' эффекты
таргеты   := id (',' id)*   |   '*'
эффекты   := эффект (',' эффект)*
эффект    := ('gold'|'acid') ':' [знак] число '%' ['*']
```

- **Несколько правил** — через `;` (можно столбиком в многострочной ячейке).
- **Таргеты** — id соседей через `,` (срабатывает, если рядом ЛЮБОЙ из них).
  Спецсимвол `*` = «все НЕназванные соседи» (вайлдкард). Смешивать `*` с
  конкретными id в одном блоке нельзя.
- **Эффекты** — через `,`. Тип: `gold` (цена) или `acid` (стоимость переваривания).
  Знак опционален (`gold:5%` = `gold:+5%`). Значение в процентах с `%`.
- **`*` в конце эффекта** = «за каждый прилегающий инстанс» (стак). Без `*` —
  применяется один раз независимо от количества соседей.

### Примеры

| Запись | Смысл |
|--------|-------|
| `sword \| gold:+50%` | рядом с мечом — цена +50% (один раз) |
| `bow \| gold:+50%*` | за каждый соседний лук — цена +50% (стак) |
| `bread \| gold:+25%; * \| gold:-25%*` | рядом с хлебом +25%; и −25% за каждый прочий сосед |
| `hat, magic_staff, anchor \| gold:-50%` | рядом с любым из трёх — цена −50% |
| `stomach \| acid:+50%` | рядом с желудком — стоимость переваривания +50% |

> Если таргетов несколько (через запятую), при экспорте Google Sheets обернёт
> ячейку в кавычки — это нормально.

## Чек-лист перед заливкой

- [ ] Порядок колонок не нарушен, лишних колонок в середине нет.
- [ ] Все `id` уникальны в своём файле.
- [ ] Все ссылки (`lootIds`, `adventurerIds`, таргеты adjacency) указывают на
      существующие id.
- [ ] Дробные числа — через точку.
- [ ] Есть хотя бы по несколько предметов `reward` и `punish`.
- [ ] У `fixture`-предметов (heart/stomach) категория = `fixture`.
- [ ] Боевые приключенцы (`battle=1`) имеют ненулевые `hp` и `attack`.
- [ ] У приключенцев с `budget>0` самый маленький предмет пула влезает в бюджет (иначе выдаст пусто).
- [ ] Формы `shape` валидны (строки одной длины, только `X`/`.`).
- [ ] Файлы сохранены как `<имя>.csv.txt` в `Game/Assets/Resources/Configs/`.
